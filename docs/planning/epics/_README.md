# Planning Epics

Epics are used only for capability-level directions that contain several related workpacks, delivery slices, or future gates.

Create an epic when:
- the work spans multiple bounded workpacks;
- the roadmap needs current, next, and later slices;
- future work must avoid becoming one giant APPLY;
- the capability needs a stable source of truth beyond a single initiative.

Do not create an epic for every bugfix, cleanup, narrow UI polish item, or one-off workpack. Those should stay as initiative/workpack artifacts unless they become part of a broader capability direction.
